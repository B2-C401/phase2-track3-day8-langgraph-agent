# HTML Demo Page Implementation Plan

**Goal:** Ship `demo/index.html` (static, single-file) + `scripts/export_traces.py` + `demo/traces.json` per spec `2026-05-11-html-demo-design.md`.

**Architecture:** Pre-recorded trace from `graph.stream()` → static JSON → vanilla JS state machine driving a hand-drawn SVG graph with CSS-animated node highlights and state-diff panel.

**Tech Stack:** Python 3.11 (script), vanilla HTML/CSS/JS (no framework, no build step).

---

## Task 1 — Write `scripts/export_traces.py`

**Files:**
- Create: `scripts/export_traces.py`

Script reads scenarios, runs each through `graph.stream()`, derives node-by-node snapshots, attaches narration, dumps to `demo/traces.json`.

Implementation details: see spec section "`scripts/export_traces.py` design". `NARRATIONS` + `SCENARIO_HEADERS` dicts embedded.

## Task 2 — Generate `demo/traces.json`

**Files:**
- Generated: `demo/traces.json`

Run `python3 scripts/export_traces.py`. Verify file size > 10 KB, 7 scenarios in output.

## Task 3 — Write `demo/index.html`

**Files:**
- Create: `demo/index.html`

Single file: inline `<style>`, body markup with header/SVG/panels/footer, inline `<script>` containing fetch + state machine + render functions.

SVG: 11 nodes with hand-tuned coordinates per spec table, edges as `<path>` elements with stable IDs.

CSS: classes `.pending/.active/.visited` for nodes, `.edge-active-pulse` for traversal, `.just-changed` for state-diff highlight.

JS: `init()` fetches `traces.json`, `render()` re-paints from current `(scenarioIdx, stepIdx)`, controls wire to Prev/Next/Play/scenario picker + keyboard shortcuts.

## Task 4 — Open in browser, verify

Open `demo/index.html` via `open demo/index.html` (file://). Click through each scenario, verify:
- All 7 selectable.
- Step counter increments.
- Active node highlights, visited nodes stay colored.
- State panel updates with `.just-changed` highlights.
- Narration text changes per step.
- Auto-play advances 1.5s per step.

## Task 5 — User commits (deferred — single commit at end)

User will batch-commit demo files with the earlier lab work. No commit during execution.
